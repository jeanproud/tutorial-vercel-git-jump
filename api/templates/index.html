Entre no site do Github e inicie o processo de criação de conta.
Selecione a opção de criar uma nova conta
Entre com Email, senha e Nome da conta
Preencha o código recebido no seu Email.
Você pode ignorar o preenchimento de personalização de conta!
Caso queira você também pode instalar o Github Desktop para utilizar os serviços do git através do aplicativo!
Entre no site do Vercel e inicie o processo de criação de conta.
Para o tipo de conta podemos escolher Hobby pois o intuito inicial e estudo! Preencha também o nome de usuário!
Agora escolha o provedor do serviço de repositório, para o nosso caso utilizaremos o GitHub!
Caso sua conta do GitHub ja esteja logada sera aberta uma janela pedindo para autorizar o link da conta do Vercel com a do GitHub
Preencha o numero de telefone para receber um código de ativação, e ative a conta com o código recebido!
Agora sera necessário adicionar o aplicativo do vercel ao GitHub!
Clique na caixa de seleção para adicionar a conta GitHub a lista!
Sera aberto uma janela do GitHub para instalarmos o app do Vercel no GitHub!
Nessa janela caso a conta GitHub tenha repositórios ativos, poderemos escolher a que repositórios o Vercel tera acesso!
Apos a aprovação sua conta GitHub aparecera na lista!
Para continuar utilizaremos um template fornecido pelo Vercel!
Esse template tera todas as configurações necessárias para o funcionamento da esteira de Deploy de repositório!
Aqui poderemos escolher um template filtrando pela linguagem ou caso de uso!
Apos confirmar visualmente o template escolhido faremos o deploy do estado inicial da aplicação!
Aqui podemos escolher o nome que daremos ao repositório que o Vercel criara em nossa conta GitHub!
O Vercel fara a configuração inicial criando o repositório, subindo o código fonte do template escolhido e fazendo o deploy da aplicação em ambiente exposta a Internet!
Apos levantar todos os recursos receberemos uma tela apresentando uma print da pagina inicial da aplicação e poderemos seguir para o dashboard!
Na tela de dashboard poderemos ver o Status do deploy, quem foi o ultimo usuário a atualizar o código fonte no GitHub, a Branch utilizada; e os links para o repositório e para a aplicação.
Alterações prontas para deploy e versão são enviadas para o repositório da forma padrão com commit de nova versão para comparação do código fonte!
Apos comparação e merge das versões podemos empurrar a nova versão para a Branch!
Assim que a nova versão da aplicação chegar no Git o aplicativo do Vercel inicia o processo de atualização exibindo quem enviou a ultima atualização!
Ao finalizar a nova versão da aplicação estará disponível e sera apresentada versão atual!
